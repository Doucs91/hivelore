## Architecture

Monorepo with four small, single-purpose packages connected by workspace dependencies:

- **`@hivelore/core`** — domain primitives only. Types, zod schemas, markdown+frontmatter parser/serializer, path resolution (`.ai/`), recursive memory loader, project-root discovery. No I/O beyond the loader. No CLI, no server.
- **`@hivelore/cli`** — `commander`-based CLI. Each subcommand lives in its own file under `src/commands/` and registers itself on the root program. The CLI never does heavy lifting itself; it delegates to core/embeddings.
- **`@hivelore/mcp`** — MCP server built on `@modelcontextprotocol/sdk` with stdio transport. Tool implementations under `src/tools/` are pure async functions taking `(input, ctx)`; `server.ts` is a thin registration layer. Embeddings are installed with the CLI/MCP packages so semantic search works after a normal global install.
- **`@hivelore/embeddings`** — optional package wrapping Transformers.js (`Xenova/bge-small-en-v1.5`, 384 dims). Lazy-loads the pipeline; exposes an `EmbedderLike` interface so tests can inject a deterministic fake instead of downloading the model.

## Key modules

- `packages/core/src/schema.ts` — single source of truth for the memory frontmatter (zod). Everything else parses through this.
- `packages/core/src/paths.ts` — `resolveHaivePaths(root)` returns every `.ai/...` path. `findProjectRoot()` walks up looking for `.ai/`, `.git/`, or `package.json`.
- `packages/embeddings/src/indexer.ts` — `rebuildIndex()` re-uses entries whose SHA-256 of `id + tags*2 + body` is unchanged.
- `packages/mcp/src/server.ts` — central registry: every tool/prompt is wired here. Adding a new tool means dropping a file under `src/tools/` and adding three lines here.

## Conventions

- ESM only. `type: "module"`, `format: ["esm"]` in tsup, `verbatimModuleSyntax + isolatedModules` in tsconfig. Imports of `.ts` source files use `.js` extensions.
- Pure tool handlers: MCP tools are unit-testable async functions; the server is just glue.
- tsup `external` lists are mandatory for any package that imports a workspace or heavy native dep.
- Context record IDs follow `YYYY-MM-DD-{type}-{slug}` and live under `.ai/memories/{personal|team|module}/`.
- Default record scope depends on project config: manual mode defaults to `personal`, autopilot defaults to validated `team` records.

## Memory types

- `convention` — coding standards, naming rules, patterns to follow
- `decision` — architecture decisions with rationale (why, not just what)
- `gotcha` — surprising behaviors, traps, non-obvious bugs
- `architecture` — structural design, module boundaries, patterns
- `glossary` — domain terminology definitions
- `attempt` — **failed approaches** ("tried X, failed because Y, use Z instead") — auto-validated, no approval needed. Use `hivelore memory tried` CLI or `mem_tried` MCP tool.

## Key features

- `mem_tried` / `hivelore memory tried` — captures failed approaches as `attempt` records (auto-validated)
- AND→OR search fallback — `mem_search` and `hivelore memory query` fall back to OR when AND returns 0 results
- `hivelore sync --inject-bridge` — injects top N context breadcrumbs into CLAUDE.md between `<!-- haive:memories-start/end -->` markers
- Briefing excludes stale by default; marks proposed as `[UNVERIFIED]`
- `mem_save` dedup warning — warns when a similar slug already exists

## Glossary

- **Anchor** — `commit + paths + symbols` recorded in a memory's frontmatter so future code edits can mark it `stale`.
- **Bridge file** — `CLAUDE.md`, `.cursorrules`, `.github/copilot-instructions.md` auto-generated by `hivelore init`.
- **Scope** — `personal | team | module`. Controls visibility and where the file lives on disk.
- **EmbedderLike** — minimal interface (`model`, `dimension`, `encode(text)`). Real `Embedder` implements it; tests pass a `FakeEmbedder`.

## Development workflow

```bash
# Build all packages
pnpm -r build

# Test hot-swap after build (avoids full npm publish cycle)
NODE_BASE=$(node -e "console.log(process.execPath.replace('/bin/node',''))")
cp -r packages/cli/dist/* $NODE_BASE/lib/node_modules/@hivelore/cli/dist/
cp -r packages/mcp/dist/* $NODE_BASE/lib/node_modules/@hivelore/mcp/dist/
# IMPORTANT: also update nested copies (see gotcha memory on nested node_modules)
cp -r packages/core/dist/* $NODE_BASE/lib/node_modules/@hivelore/cli/node_modules/@hivelore/core/dist/

# Run tests
pnpm -r test

# Version bump + publish
# 1. Bump all package.json to same version
# 2. pnpm -r build
# 3. git commit + git tag vX.Y.Z
# 4. npm publish --access public in each package dir
```

## Gotchas

- Without explicit `external` in `tsup.config.ts`, tsup will inline `@xenova/transformers` + `onnxruntime` and explode the CLI bundle past 5MB.
- `gray-matter` parses YAML date strings as `Date` objects. The `IsoDateString` zod helper normalizes both.
- `gray-matter` / js-yaml refuses to serialize `undefined` values. `serializeMemory` strips them recursively.
- `@hivelore/embeddings` must ship with `@hivelore/cli` and `@hivelore/mcp`; if it is only a peer dependency, global installs lose semantic ranking and code_search.
- The MCP server logs to stderr — never to stdout, since stdout is the JSON-RPC channel.
- `require.resolve("@hivelore/mcp/package.json")` requires `"./package.json"` in the exports field of `@hivelore/mcp`.
