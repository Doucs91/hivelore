# Project context ŌĆö hAIve

> Team-first persistent memory layer for AI coding agents. Built with TypeScript, published as a pnpm workspace monorepo.

## Architecture

Monorepo with four small, single-purpose packages connected by workspace dependencies:

- **`@haive/core`** ŌĆö domain primitives only. Types, zod schemas, markdown+frontmatter parser/serializer, path resolution (`.ai/`), recursive memory loader, project-root discovery. No I/O beyond the loader. No CLI, no server.
- **`@haive/cli`** ŌĆö `commander`-based CLI. Each subcommand lives in its own file under `src/commands/` and registers itself on the root program. The CLI never does heavy lifting itself; it delegates to core/embeddings.
- **`@haive/mcp`** ŌĆö MCP server built on `@modelcontextprotocol/sdk` with stdio transport. Tool implementations under `src/tools/` are pure async functions taking `(input, ctx)`; `server.ts` is a thin registration layer. Embeddings are pulled via `import("@haive/embeddings")` (optional dep) so the server runs even when embeddings are absent.
- **`@haive/embeddings`** ŌĆö optional package wrapping Transformers.js (`Xenova/bge-small-en-v1.5`, 384 dims). Lazy-loads the pipeline; exposes an `EmbedderLike` interface so tests can inject a deterministic fake instead of downloading the model.

## Key modules

- `packages/core/src/schema.ts` ŌĆö single source of truth for the memory frontmatter (zod). Everything else parses through this.
- `packages/core/src/paths.ts` ŌĆö `resolveHaivePaths(root)` returns every `.ai/...` path. `findProjectRoot()` walks up looking for `.ai/`, `.git/`, or `package.json`.
- `packages/embeddings/src/indexer.ts` ŌĆö `rebuildIndex()` re-uses entries whose SHA-256 of `id + tags*2 + body` is unchanged.
- `packages/mcp/src/server.ts` ŌĆö central registry: every tool/prompt is wired here. Adding a new tool means dropping a file under `src/tools/` and adding three lines here.

## Conventions

- ESM only. `type: "module"`, `format: ["esm"]` in tsup, `verbatimModuleSyntax + isolatedModules` in tsconfig. Imports of `.ts` source files use `.js` extensions.
- Pure tool handlers: MCP tools are unit-testable async functions; the server is just glue.
- tsup `external` lists are mandatory for any package that imports a workspace or heavy native dep ŌĆö see `gotcha-tsup-externals-required`.
- Memory IDs follow `YYYY-MM-DD-{type}-{slug}` and live under `.ai/memories/{personal|team|module}/`.
- v0.1 follows "Personal first": new memories default to `personal`; promotion to `team` is explicit (`haive memory promote <id>`). v1.0 will switch to "Hybrid intelligent" classification.

## Glossary

- **Anchor** ŌĆö `commit + paths + symbols` recorded in a memory's frontmatter so future code edits can mark it `stale`.
- **Bridge file** ŌĆö `CLAUDE.md`, `.cursorrules`, `.github/copilot-instructions.md` auto-generated by `haive init` to point non-MCP-aware AI tools at `.ai/project-context.md`.
- **Scope** ŌĆö `personal | team | module`. Controls visibility and where the file lives on disk.
- **EmbedderLike** ŌĆö minimal interface (`model`, `dimension`, `encode(text)`). Real `Embedder` implements it; tests pass a `FakeEmbedder` that returns deterministic vectors per keyword.

## Gotchas

- Without explicit `external` in `tsup.config.ts`, tsup will inline `@xenova/transformers` + `onnxruntime` and explode the CLI bundle past 5MB. Always list workspace deps and heavy native packages.
- `gray-matter` parses YAML date strings as `Date` objects. The `IsoDateString` zod helper in `core/src/schema.ts` normalizes both `Date` and `string` to ISO strings before piping to `z.string().datetime()`.
- `gray-matter` / js-yaml refuses to serialize `undefined` values. `serializeMemory` in `core/src/parser.ts` recursively strips them.
- `@haive/embeddings` is an `optionalDependency` of `@haive/cli` and `@haive/mcp` ŌĆö without it declared, the `import("@haive/embeddings")` in `mem-search.ts` silently fails and you get `mode: "literal_fallback"` instead of semantic results.
- The MCP server logs to stderr (`[haive-mcp] startingŌĆ”`) ŌĆö never to stdout, since stdout is the JSON-RPC channel.
